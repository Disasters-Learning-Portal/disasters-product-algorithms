# =============================================================================
# Disasters Hub managed conda deps
# =============================================================================
# Source of truth for conda packages that the Disasters Hub image
# (pangeo-notebook-veda-image) needs ON TOP OF the Pangeo base image.
#
# Format: one conda package spec per non-comment line, exactly as it would
# appear under `dependencies:` in environment.yml (e.g. "gdal" or "rasterio>=1.3").
# Lines starting with # are ignored. Blank lines are ignored.
#
# Changes here are auto-synced to pangeo-notebook-veda-image's environment.yml
# via .github/workflows/sync-conda-deps.yml: on push to main that touches this
# file, a PR is opened in the image repo replacing the contents of the
# `# === BEGIN disasters-product-algorithms managed conda deps ===` block.
# Review and merge that PR — the next image build then picks up the new
# conda dep (the algorithms install layer is independent and unaffected).
#
# Pip-first preference:
#   If a new dep ships a manylinux wheel, prefer adding it to
#   pyproject.toml's `[project] dependencies` instead. Those install
#   transitively via the existing `pip: - git+https://...algorithms.git`
#   line in the image repo's environment.yml — no auto-PR needed, no
#   touch of the image repo at all. Only add to THIS file when a package
#   is conda-only (e.g., a binary system library or a specific GDAL plugin)
#   and is not already present in the Pangeo base image.
#
# For DEV-LOCAL conda install, see the conda-dep comment block in pyproject.toml.
# =============================================================================
